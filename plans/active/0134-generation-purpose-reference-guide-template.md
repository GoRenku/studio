# Generation Purpose Reference Guide
Audio purposes migrate to the new generic generation contract, but their product
requirements do not otherwise change. They are not expanded below because they
did not have dependency/reference guidance to revise.

## Rules That Apply Automatically

- every listed reference is guidance, not a required dependency unless it is required by the provider model (like first image for example)
- Studio never generates a missing reference and has no smarts about reference
  contents. The only automatic selections are the explicit first-sheet UI
  defaults documented below for Profile, Hero, and Storyboard Sheet; users and
  agents may replace them.
- every generation also allows arbitrary Additional adhoc References;
- the selected provider model alone determines what must be present before the generation can run.

## Some general behaviors:
- Values marked Required are fixed Studio-owned settings because changing them
  would break Studio presentation or deterministic product workflows. Values
  marked Recommended are guidance and initial UI defaults; users and agents may
  change them.
- In the UI (Preview Generation and Regenerate dialogs, AI Production references tab) when we show a typed (purpose) reference, it acts as a slot for that type. If there are multiple options (resolved images etc.), then it pops up a dialog for the user to select the desired option. This is already implement in AI Production/Reference tab. This should be a general and reusable behavior/component with the same styling and UX paradigm.
  - E.g. selecting a character-sheet slot with multiple options, we should pop up a dialog for the user that they can select from the available character sheets (filtered by the character of course)
- For each of the below purposes, the generation can happen using an external tool and then attached. Separation of generation from attachment to the right concept in the app is very important. Users may often want to use their favorite subscriptions etc. to do the generations and then bring it back here to attach them.
- The relief valve for generations is the ability to add an ad-hoc reference
  image. AI generation is an ever-changing field, so users use many techniques.
  - A reference generated by Renku has its own `image.create` spec and is
    explicitly attached to the later work.
  - A user-supplied or externally generated reference has no Renku generation
    spec. It is attached directly to the owning target/work item and is not
    reusable elsewhere unless the user separately imports it into a reusable
    project domain.
- For image generation, Studio shows this explicit fal.ai model allowlist rather
  than every model in the Engines catalog:
  - gpt-image-2 (also available from Codex and preferred for text-only generations)
  - nano-banana-2
  - nano-banana-pro (We don't have this available so we need to make it)
  - xai-grok imagine image 1.5
  - seedream 5 pro (We don't have this available yet, next iteration)
  - More to be added later
- The image generation models should be available in the drop-down in the dialogs (Generation Preview and Regerated/Edit Dialog) and selectable by users. Edit always uses the edit (or reference) versions of the models. The generate can use text-to-image or reference-to-image versions depending on the particular purpose.
- The video generation model options are unchanged and already inplace

## lookbook.sheet -> lookbook.video-sheet, lookbook.storyboard-sheet
This is a sheet that summarizes the main aspects of the lookbook. It can be used in 2 different purposes: video and storyboard. Therefore going forward we should separate this purpose into 2: lookbook.video-sheet, lookbook.storyboard-sheet

- They are global for the project.
- There can be multiple sheets and users (agents) can choose the appropriate ones when using them in another generation. There is no default etc.
- The videos use the sheets that are for the final production (or previs). The storyboards are mainly for visualizing the story and they usually have a more hand-drawn style.
- Recommended:
  - aspect ratio is 4:3
  - quality is high
  - model is gpt-image-2

When generating:
- These are generated from the lookbook text and also special direction (if any) given by the user during generation.
- They can be generated text-to-image or reference-to-image by providing
  Lookbook Images or ad-hoc images.

In the UI:
- Preview Generation and Regeneration Dialogs
  - Should show adhoc references, lookbook images
- Visual Content Tab
  - Rename it to Assets to be consistent with Cast members section

## lookbook.image
These are sample images generated to demonstrate various aspects of the lookbook and then attached/tagged so that the lookbook page in Studio app can show these images inline with the lookbook in the correct locations.

- Recommended
  - aspect ratio is the same as project one
  - quality is medium
  - model is nano-banana-2

The edit dialog options are not available for these in the UI at the moment, probably later. Preview Generation dialog should be available though.

## cast.character-sheet -> cast.video-character-sheet, cast.storyboard-character-sheet
This is for character consistency. It can be used for videos and storyboards. Therefore going forward we should separate this purpose into 2: cast.video-character-sheet, cast.storyboard-character-sheet to make the distinction clear.

- They are global and declared per cast member.
- There can be many character-sheets per cast member. (E.g. with different costumes, ages, looks ...) And also multiple for video vs. storyboard purposes.
- The videos use the sheets that are for the final production (or previs). The storyboards are mainly for visualizing the story and they usually have a more hand-drawn style.
- Recommended:
  - aspect ratio is 16:9
  - quality is high (because it needs to provide details of face etc.)
  - model is gpt-image-2 which provides the best for these types.
When generating:
- First character-sheet will most likely be generated using: (choice left to agent or user)
  - text only description (text-to-image) that describes both the character and the lookbook style in text. To do this agent will still need to get the lookbook text as context to be able to create the prompt properly.
  - or by providing references to some actual images (like a historical character) and as well as a lookbook video or storyboard sheet for look consistency.
- The user will specify to the agent whether it is for storyboard vs. video. It is then up to the agent to get the context and use the correct lookbooks and instructions to generate for either of those.
  - Since the distinction is obvious in the UI (e.g. in assets tab, generation preview or regenerate dialogs) there is no need to specify this in a separate list. Users can visually distinguish the character sheets anyways.
- Later generated character sheets for a cast member will likely use other character-sheets for character consistency in generation.

In the UI:
- Preview Generation and Regenerate dialogs:
  - Should show ad-hoc references, lookbook sheets or character sheets. Users can select/deselect those
- Assets Tab (Cast member)
  - Like today no changes

## cast.profile
This is mainly for the Studio UI to provide a recognizable character image across the product. Will likely not be used in other generations.
- There is one in the system per cast-member, the Studio UI can deterministically access it wherever it needs it.
- If re-generated or edited then it takes place as the canonical profile image.
- It has a specific profile image look, which means the character is shown from the shoulders up (See existing character profile images in the sample urban-basilica project)
- Required: aspect ratio is square (1:1)
- Recommended:
  - quality: medium
  - model: nano-banana-2 (for cost)
When generating:
- It is generated from a video Character Sheet and defaults to the first one.
- Preview Generation and Regenerate/Edit show that selected slot. Users can
  choose another available video Character Sheet through the shared
  multiple-choice reference dialog.

## location.environment-sheet -> location.sheet
Used for defining the environment in more detail for the video and used for location consistency

- They are global and declared per location
- There can be many sheets for a location
- Recommended:
  - aspect ratio is 16:9
  - quality is high
  - model is gpt-image-2

When generating:
- Similar to the cast members with the same behavior but substitute character with location

In the UI:
- Preview Generation and Regenerate dialogs:
  - Should show ad-hoc references, lookbook sheets or character sheets. Users can select/deselect those
- Assets Tab (Location) - rename from Visual Content for consistency
  - Like today no other changes

## location.hero
Same idea as the cast.profile but for location. All applies here except:
Required:
- Aspect Ratio is 16:9

## scene.storyboard-sheet
Generates a sheet of 2x2 images that depicts the shots in a scene. Used for storyboarding only, not intended for use in actual shots for video generation

- Internal optimization technique for Renku Studio so that it generates 4 images at once. Is not configurable by user
- The agent then splits the sheet into four images using the current Skill
  instructions. This golden workflow must not be replaced by a runtime splitting
  library or automatic Studio/Core/Engines splitter.
- Required:
  - overall sheet aspect ratio: 4:3
  - each image in the 2x2 grid uses the project aspect ratio
  - quality: high
- Recommended
  - model is gpt-image-2
- It should by default use the storyboard lookbook. If not yet created, warn the user that it is not created. Do not make up something, because consistency between sheets are key. They should not be adhoc

Essentially no changes here.

## shot.video-take
Generates the video for the shot or collection of shots in the storyboard.

When generating:
- References are grouped into
  - Shot-owned references.
    - Depending on the input type of the video model (First frame, First/Last Frame etc.) there will be some required references (by the model provider).
      - First Frame -> First Frame image needs to be present
      - First/Last Frame -> Both first and last frame needs to be present
      - Video Prompt Sheet -> These are popular (but optional) way to specify multi-shot generations by providing a sheet with each shot represented in a grid. Different than the storyboard-sheets in that they are intended for "accurate" video take generation.
      - Reference -> Here no specific requirement but lack of references should be a warning during generation time.
    - All these references are generated using image.create purpose
    - Users can provide ad-hoc references by generating them through
      `image.create`; that earlier generation keeps its own spec and the result
      is then attached as an exact reference.
    - Users can also attach externally supplied images. Those files have no
      Renku generation spec and must not receive synthetic generation provenance.
    - Users can also provide dialog audio (as it is now)
  - References to various recommended consistency providing project level purposes
    - Cast character sheets
    - Location sheets
    - Lookbook sheets (notice that this can also be alternatively specified as part of the prompt by the AI agent. User can choose their preference when prompting the agent)

UI
- Regenerating these are done using the Takes-Edit tab. Already implemented and no changes
- There is currently no Edit UI available (This will use video editing models etc.) TBD
- The preview generation dialog is available and must be popped up by the agent so that the user can review

## image.edit
This is a special one only used for editing images.

- Always provides the existing image as the reference so it can be edited.
- Uses the image edit models
- Prompts are specifically for modifying an existing image.
- It may have other references:
  - Fully adhoc image references (generated or simply provided such as photos etc.)
  - References to existing character sheets, prop sheets (in the future when we have them), location sheets
  - References to a lookbook sheet so the look of the image can match a different one.

UI:
- The new Edit tab of the Regenerate/Edit should always use this.
- The agent AI should use this through skill instructions when asked to make modifications to a generated or existing image.
