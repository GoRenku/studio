# Renku Studio Documentation

This folder is for accepted project knowledge: product vocabulary,
architecture, UI direction, operations notes, and decision records.

Use these sections by document role:

- `architecture/` contains current architecture overviews and implementation
  topic maps.
- `architecture/reference/` contains precise current architecture contracts and
  implementation references.
- `cli/` contains the living command reference for the `renku` CLI.
- `decisions/` contains accepted ADRs. Use ADRs for durable decisions and
  historical context about why an architecture choice was made.
- `governance/` contains documentation taxonomy and audit records.
- `product/`, `ui/`, and `operations/` contain accepted product, interface, and
  operating guidance.
- `plans/active/` contains current implementation plans and sequencing.
- `plans/exploration/` contains unresolved or historical exploration.

When a plan or exploration becomes accepted direction, promote the durable
decision into `docs/decisions/` and keep `docs/architecture/` as the readable
overview that points to those ADRs.
